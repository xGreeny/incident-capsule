## Trustworthy finalization and safe handoff

Incident Capsule 1.1.0 makes acquisition readiness, collection limits, coverage gaps, and handoff integrity explicit without expanding the tool into remote response or remediation.

### Highlights

- Preflight the intended run with `Test-IncidentCapsuleReadiness`.
- Bound capsule size, EVTX files, native commands, native output, high-volume inventories, and the derived timeline.
- Review structured coverage and machine-readable limitations in `metadata/coverage.json`.
- Navigate timestamped evidence through the bounded `analysis/timeline.json` and `.csv` indexes with source provenance.
- Distinguish collection state from terminal finalization state.
- Verify directories and archives fail-closed, including sidecars, checksum lists, ZIP paths and entry types, extraction quotas, and external verification receipts.
- Use self-contained release packages smoke-tested in Windows PowerShell 5.1 and PowerShell 7.

See the changelog and `docs/readiness-and-handoff.md` for configuration and operational guidance.
