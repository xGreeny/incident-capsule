## Field-hardening from a production run

Incident Capsule 1.2.1 is a maintenance release driven by a real first-response collection on Windows Server 2025 under Windows PowerShell 5.1.

### Highlights

- The System collector no longer fails completely under Windows PowerShell 5.1 (`Environment.TickCount64` is .NET Core only); host identity, boot, clock, and execution-context evidence is collected again on default server shells.
- The ScheduledTasks collector survives tasks without `CimClass` metadata and now isolates each task, so one malformed task cannot erase the remaining task evidence.
- Logon-session and AppCompatCache edge cases produce precise warnings instead of avoidable failures.
- `capsule.json` records the collecting PowerShell version and edition; `channels.json` records per-channel query and export durations so slow event channels are visible.
- The CI smoke test now fails on any collector-level failure, and new regression tests exercise the affected collectors in both PowerShell editions.

Update recommended for every deployment that collects with Windows PowerShell 5.1.
