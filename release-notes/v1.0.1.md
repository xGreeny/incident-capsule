## Integrity and lifecycle hardening

Incident Capsule 1.0.1 strengthens the trust boundary around generated and received evidence packages without expanding collection scope.

### Highlights

- Rejects absolute, traversing, duplicate, alternate-data-stream, and reparse-point paths in manifests and archives.
- Inspects ZIP entry count, individual size, expanded size, compression ratio, and entry type before extraction.
- Verifies a newly created archive before `-RemoveWorkingDirectory` can delete source evidence.
- Writes an external machine-readable verification receipt beside every generated archive.
- Uses atomic UTF-8 writes for structured output and spreadsheet-safe derived CSV exports.
- Enforces hard collection and verification limits in addition to profile defaults.
- Pins build dependencies and validates the packaged module before release.

Raw JSON remains the canonical evidence representation. CSV files are derived analyst views.
