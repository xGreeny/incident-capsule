## Evidence depth and authentic handoff

Incident Capsule 1.2.0 adds four read-only collectors for classic triage artifacts, cryptographic manifest signing, SIEM export, and PowerShell Gallery distribution — without expanding the tool beyond local, bounded, read-only acquisition.

### Highlights

- Collect installed software from the uninstall registry keys, the local-machine certificate trust stores, execution artifacts (bounded prefetch copies, raw Shimcache, BAM, UserAssist), and removable-device history (USBSTOR, MountedDevices, MountPoints2, bounded `setupapi.dev.log`).
- Seal the checksum list with a detached CMS signature via `-SigningCertificate` and require it at handoff with `Test-IncidentCapsuleIntegrity -RequireSignature`; signer details land in integrity results and verification receipts.
- Export capsule evidence as line-delimited JSON with `Export-IncidentCapsuleData` for Splunk, Sentinel, or Timesketch — written beside the capsule, never into it.
- Watch lateral-movement channels in the Extended profile: WinRM, PrintService, and OpenSSH operational logs.
- Install directly from the PowerShell Gallery: `Install-Module IncidentCapsule`.

Schema version is now 1.2; capsules written with schema 1.0 and 1.1 remain verifiable. See the changelog and `docs/collector-reference.md` for the complete output contract.
