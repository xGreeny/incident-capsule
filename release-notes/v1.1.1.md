## Engine hardening and expanded test coverage

Incident Capsule 1.1.1 is a maintenance release. It hardens two orchestration edge cases and expands the unit test suite; collection behavior and the public interface are unchanged.

### Highlights

- A fatal error before the first collector completes is no longer masked: `Get-ICOverallStatus` now accepts an empty collector result set, so finalization reports the original failure.
- A collector output file outside the capsule root is now recorded in `collector.log` in addition to the collector warnings.
- New unit tests cover collector engine failure paths: terminating errors, missing result objects, out-of-root output files, unregistered collectors, and overall-status computation.
- New unit tests cover offline report generation: empty collector results, HTML encoding of evidence-derived values, metric rendering, and unsafe evidence-link omission.
- Capsule directory and archive name examples in the documentation now match the actual naming scheme, which includes a random suffix.

See the changelog for details.
