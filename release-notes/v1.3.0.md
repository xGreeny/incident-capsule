## Capsule comparison and event-collection tuning

Incident Capsule 1.3.0 adds baseline-versus-incident comparison and tightens event collection based on a production run.

### Highlights

- `Compare-IncidentCapsule` diffs two capsules of the same host and reports added, removed, and changed records across services, scheduled tasks, installed software, autorun registry values, local users and groups, certificate trust stores, and system drivers. Volatile evidence (processes, live network endpoints, event summaries, timeline) is intentionally excluded so real persistence-, account-, and inventory-level change stands out. The comparison report is written outside both sealed capsules.
- Decoded event summaries respect a new `MaximumEventMessageChars` bound, so a single noisy channel (for example PowerShell script-block logging) no longer inflates the capsule; native EVTX exports remain complete and each truncated message is flagged.
- Optional event channels (Sysmon, OpenSSH, PrintService) no longer mark a collection partial when they are absent; their absence is reported as informational coverage in both `channels.json` and readiness checks.

Schema version is now 1.3; capsules written with schema 1.0 through 1.2 remain verifiable. See the changelog and `docs/collector-reference.md` for details.
